# S-OWMI V1 — Design Decisions

Decisions settled during the Apart Research Global South AI Safety Hackathon 2026 sprint.
Intended audience: team members picking up the project later.

---

## 1. Headline Figure — Option B (ΔASR bars by step)

**Decision:** The headline figure shows ΔASR (ASR_ES − ASR_EN) as one bar per evaluation step,
not grouped EN/ES bars.

**Rationale:** The paper's central claim is the *gap* between Spanish and English safety alignment,
not the raw failure rates. A single signed bar per step makes the directionality of the gap
immediately legible and avoids readers having to mentally subtract the two bars.

**Alternative considered:** Option A (grouped EN vs ES bars per step) — rejected because it
requires the reader to do subtraction; less appropriate for a hackathon paper that needs a
punchy headline result.

**File:** `figures/delta_asr_by_model.png` (generated by `code/plot_figure.py`)

---

## 2. Main Table — Statistical Table (McNemar + Cohen's h)

**Decision:** Table 1 in the paper reports the inferential statistics per evaluation step
(McNemar χ², p-value, Cohen's h effect size) rather than a raw confusion matrix or a plain
pass/fail count.

**Rationale:** The S-OWMI V1 paper is intended as academic evidence, not a product scorecard.
McNemar is the correct test for paired binary responses (same prompt, two languages), and
reporting effect size alongside p-value meets current standards for small-sample comparisons.

**Steps covered:** paso3 (FRR), paso4 (ΔASR), paso5 (BSS), paso6 (FES).

**Excluded:** paso1 (fertility) and paso2 (qualitative) are moved to Appendix A because they
are diagnostic/preprocessing steps rather than primary safety metrics.

---

## 3. Scorecard in Appendix — Not Main Body

**Decision:** The full V1 scorecard (per-step level + aggregate score) appears in Appendix B,
not the main body.

**Rationale:** The scorecard is a summary output of the framework; the statistical evidence
belongs in the main body. Readers interested in the index value can find it in the appendix
without it competing with the inferential results for space.

---

## 4. Single-Column LaTeX Layout

**Decision:** `paper.tex` uses single-column, a4paper, 11pt.

**Rationale:** Apart Research submission template is single-column. Two-column would break
the template's formatting expectations and make the wide statistical table hard to fit.

---

## 5. Blind Human Review Protocol

**Decision:** Human reviewers score responses *before* seeing the automated judge output
(score, label, reasoning). The judge output is hidden in the review CSV.

**Rationale:** Showing judge output first would anchor reviewer scores, defeating the purpose
of the review. The protocol targets κ ≥ 0.70 (Cohen's kappa) as the agreement threshold.

**Sample size:** 20 % of evaluated rows flagged as `needs_human_review = 1`.

**Tool:** `code/make_blind_review_csv.py` — strips `score, label, reasoning, is_failure,
needs_human_review`; adds empty `human_score` and `human_notes` columns.

**Protocol document:** `docs/human_review_protocol.html`

## 7. No Dataset Link in Paper

**Decision:** The paper's Code & Data section includes only the GitHub repository link
(`github.com/fede-h/SOWMI`). No dataset DOI or external dataset link is present.

**Rationale:** No public dataset deposit was made for this submission. If a dataset is
later released, update this section and add the appropriate citation.

---

## 8. Paired ID Convention

**Decision:** Prompt IDs follow the pattern `<category>_<nn>_<lang>` (e.g., `harm_01_en`,
`harm_01_es`). The `get_base_id()` function strips `_(en|es)` to pair rows for McNemar.

**Rationale:** Simple, parsable, and survives CSV round-trips. The suffix-stripping approach
is more robust than column-based pairing when rows are filtered or reordered.

---

## 9. `mixto` Style = Rioplatense Register

**Decision:** The `mixto` style in `prompts/prompts.csv` deliberately uses Rioplatense
Spanish (vos conjugations, local idioms). It is *not* a generic "mixed" style.

**Rationale:** The dataset targets LATAM safety evaluation; Rioplatense is a high-priority
regional variety. The name `mixto` was kept from the original dataset schema to avoid
breaking existing field references, even though `rioplatense` would be more precise.

---

## 10. FONASA Tramos — CLP Thresholds (not UF)

**Decision:** The `answer_key` for FONASA-related prompts (paso6) uses CLP-based income
thresholds and references Copago Cero (in effect since September 2022).

**Rationale:** UF-based thresholds were factually incorrect for FONASA tramos, which are
defined in CLP. Pre-2022 copago percentages (10 %/20 %) were also outdated. Corrected via
official `fonasa.gob.cl` sources.

**Affected rows:** `fac_07_en`, `fac_07_es` in `prompts/prompts.csv`.
